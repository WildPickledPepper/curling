# 《The Motion of a Curling Rock》模型分析

## 1. 结论先行

这篇 1996 年论文不是一个可以直接运行的服务器模拟器，也没有给出源代码。它给出的是一套可数值积分的自由滑行物理模型：冰壶底部接触环同时受到干摩擦和水膜湿摩擦，二者在冰壶减速过程中按三个阶段改变空间分布，从而产生正确方向的弯曲。

它适合用作我们本地模拟器的**物理结构先验**，但不能直接声称复刻了官方服务器，原因是：

- 论文只研究单壶无碰撞滑行；
- 论文没有给出完整的 sweeping 动力学方程；
- 论文参数来自真实冰面，不一定等于数字冰壶服务器参数；
- 官方协议中的 `BESTSHOT(v0,h0,w0)`、`MOTIONINFO` 坐标和内部时间步仍需实验确认。

因此最合理的路线是：保留论文的状态方程和摩擦结构，再用官方采样拟合参数，而不是直接照抄论文参数。

## 2. 模型状态

自由滑行可以写成连续时间状态：

```text
s(t) = [x, y, vx, vy, omega]
```

其中 `(x,y)` 是质心位置，`(vx,vy)` 是质心速度，`omega` 是自转角速度。动力学为：

```text
dx/dt     = vx
dy/dt     = vy
dvx/dt    = Fx / M
dvy/dt    = Fy / M
domega/dt = tau / J
J         = M R^2 / 2
```

`Fx`、`Fy` 和 `tau` 不是常数，而是接触环各位置摩擦力积分后的合力与力矩。

## 3. 接触环与局部速度

论文将冰壶底部的窄接触环分成前后半圆、内外区域和四个象限。接触环半径约 `R=0.125 m`，宽度约 `3-6 mm`。

接触环角度 `theta` 处相对冰面的局部速度是平动和转动之和：

```text
u(theta) = v_cm + omega x r(theta)
```

每个微元的摩擦方向都与 `u(theta)` 相反。由于转动速度在接触环不同位置方向不同，各位置的摩擦方向也不相同。

## 4. 两类摩擦

干摩擦采用近似库仑摩擦：

```text
dF_d = mu * M * g * dtheta / (2*pi)
```

运动产生的热量会形成薄水膜，湿摩擦写成速度的幂函数：

```text
dF_w = k * |u(theta)|^phi
```

论文数值示例使用：

```text
mu     = 0.01
k      = 0.2 kg/m
phi    = 2
DeltaR = 6 mm
```

这些值只能作为初始化参数，不能未经官方数据验证就固定到我们的模拟器中。

## 5. 三阶段运动

论文模型最有价值的部分不是某一个系数，而是随速度改变的摩擦分区。

### 第一阶段：高速

- 前部外侧主要为干摩擦；
- 前部内侧和后部主要为湿摩擦；
- 平动速度和转速下降较快；
- 横向弯曲较小。

### 第二阶段：中速

- 水膜有一部分在后部经过前已经重新冻结；
- 前后部的干湿摩擦不对称较弱；
- 轨迹相对较直，但仍保持正确方向的小曲率。

### 第三阶段：低速

- 旋转的冰壶会把水膜从后部拖带到前部；
- 前部大部分变湿，后部大部分变干；
- 横向合力明显增大，终段 curl 被放大。

论文让前部干摩擦比例随速度平滑变化：

```text
v >= 1.5 m/s : 约为 1
v ~= 1.0 m/s : 约为 1/2
v -> 0       : 约为 0
```

每次过渡的典型平滑宽度约为 `0.2 m/s`。实现时可用 logistic 或 smoothstep 表达这些比例。

## 6. 数值结果与现实量级

论文示例中：

- draw：初速度 `3 m/s`，初始角速度 `2*pi/3 rad/s`；
- takeout：初速度 `4 m/s`，初始角速度同上；
- draw 约运行 `28 s`、前进约 `28 m`；
- 总横向 curl 约 `0.4 m`，约为 `1.5` 个壶直径；
- 大约一半横移发生在最后 `6 m`；
- 相同平动速度下，更大的初始转速产生更大的 curl，但前进距离接近。

模型复现了一个关键非线性特征：**curl 不是从出手到停止均匀积累，而是低速末段显著增强。**

## 7. Sweeping 能告诉我们什么

论文明确说明其计算轨迹不包含 sweeping，因此没有可直接实现的 `sweep -> 位移` 公式。它只提出了机制和可检验预测：

- sweeping 形成额外水膜、降低局部干摩擦，因此通常延长滑行距离；
- 在冰壶特定一侧 sweeping，可能通过改变前后干湿摩擦分布增强终段 curl；
- 另一侧 sweeping 对横向合力的影响可能很小；
- 文中讨论一次较强 sweeping 可能让过轻的壶多走约 `4 m`，但这不是通用线性系数。

这与当前 `fast_curling_env.py` 中的

```python
y = y - sweep * 0.045
```

有本质区别。当前实现把 sweep 当作固定线性纵向位移，既不依赖速度、旋转和擦冰时机，也没有横向作用。官方 sweep 样本已经表现出明显非线性，因此这条线性公式不应作为最终模型。

## 8. 与当前本地模拟器的差距

| 能力 | 当前实现 | 论文模型能否改善 |
| --- | --- | --- |
| 无碰撞最终落点 | 12 项多项式回归 | 可提供外推更合理的动力学结构 |
| 完整运动轨迹 | 没有 | 可以通过 ODE 积分产生 |
| 终段 curl 增强 | 回归隐式拟合 | 可以显式表达 |
| sweep | 固定线性纵向偏移 | 只提供机制，仍需官方采样拟合 |
| 碰撞 | 最近壶固定推移 | 不能，需另建刚体碰撞模型 |
| 出界/规则 | 简化边界判断 | 不能，仍由规则层处理 |

目前无 sweep 落点回归在采样范围内验证 RMSE 约 `0.038 m`，因此不应立刻被论文模型替换。论文模型首先应该作为并行实验模块，在留出数据上与现有回归比较。

## 9. 建议的可校准实现

### 9.1 第一版 ODE

1. 状态使用 `[x,y,vx,vy,omega]`；
2. 将接触环离散成 `32-128` 个角度点；
3. 每个点计算局部速度和干/湿摩擦；
4. 按速度平滑计算三阶段干湿比例；
5. 求和得到 `Fx,Fy,tau`；
6. 用 RK4 或小步长半隐式 Euler 积分；
7. 当速度和角速度低于阈值时停止。

### 9.2 待拟合参数

```text
mu, k, phi
有效接触半径 R 与环宽 DeltaR
两个阶段切换速度及平滑宽度
各阶段前后/内外干湿比例
BESTSHOT 到 [vx0, vy0, omega0] 的映射
停止阈值和可能的随机噪声
```

### 9.3 目标函数

只有最终落点时可先拟合：

```text
L_land = mean(||p_pred_final - p_official_final||^2)
```

若 `MOTIONINFO` 能稳定提供中间速度或位置，则加入轨迹、速度和停止时间约束。仅凭最终落点很难唯一识别全部物理参数；多组参数可能得到相近终点。

## 10. 对项目的直接判断

这篇论文值得用，但用法应当克制：

1. 当前官方范围内的落点精度仍由数据回归兜底；
2. 新建论文 ODE 模块，先只处理无碰撞、无 sweep；
3. 用相同训练/验证划分比较回归模型与 ODE 模型；
4. 如果 ODE 在范围内不输、范围外更稳定，再接入搜索；
5. sweep 作为条件摩擦参数单独拟合；
6. 碰撞另建基于质量、恢复系数和切向摩擦的二维刚体模块。

最重要的收益不是把模拟器突然变得“更真实”，而是让我们从只会预测终点的黑箱，升级成能预测整个运动过程、能够解释终段弯曲、也更容易承载 sweep 的状态转移模型。

## 11. 已实现原型与实测结果

当前已经实现：

- `paper_curling_sim.py`：从中线状态积分到停止的批量 ODE 模拟器；
- `tools/calibration/fit_paper_physics.py`：拟合出手到中线映射及动力学参数；
- `config/paper_physics_calibration.json`：拟合参数、映射系数和评估报告；
- `tests/test_paper_curling_sim.py`：静止、旋转对称和配置加载测试。

复现实验：

```powershell
D:\anaconda3\python.exe tools/calibration/fit_paper_physics.py data/calibration/no_sweep_200.jsonl --dt 0.0125
D:\anaconda3\python.exe -m unittest discover -s tests -p "test_*.py" -v
```

使用 151 条训练样本、39 条固定验证样本：

| 模型 | 验证 RMSE |
| --- | ---: |
| ODE，使用官方中线状态 | 0.03719 m |
| ODE，预测中线状态的完整管线 | 0.04032 m |
| 直接终点多项式，同训练分割 | 0.03751 m |
| 回归 x + ODE y 的 hybrid | 0.03708 m |

这表明 ODE 尾程本身能够解释官方数据，纵向分量有小幅收益；完整 ODE 的主要损失来自出手到中线映射及横向 curl。hybrid 的优势约为 0.43 mm，样本量尚不足以证明稳定优势，因此当前默认比赛模拟器不切换。

拟合得到的代表性参数包括：

```text
dry_decel       = 0.07165 m/s^2
wet_decel       = 0.01921
wet_exponent    = 2.534
transition_speed = 1.158 m/s
curl_gain       = 0.000870
late_curl_gain  = 4.818
```

其中湿摩擦指数接近论文的 `phi=2`，阶段切换速度也落在论文讨论的约 `1.0-1.5 m/s` 区间，但这些是数字服务器的有效参数，不能解释为真实冰面的直接测量值。

## 12. 全量部署重拟合

完成模型选择后，部署配置又用全部样本进行了重拟合：

- 200/200 条样本均有中线 `MOTIONINFO`，全部用于出手到中线映射；
- 190/200 条样本有真实最终坐标，全部用于 ODE 尾程参数；
- 10/200 条出界样本只返回 `(0,0)`，这是删失标记而非真实落点，不能作为坐标损失。

评估配置和部署配置分别保存在：

```text
config/paper_physics_evaluation.json
config/paper_physics_calibration.json
```

全量修正后的主要参数：

```text
dry_decel        = 0.07272 m/s^2
wet_decel        = 0.01797
wet_exponent     = 2.622
transition_speed = 1.235 m/s
curl_gain        = 0.000767
late_curl_gain   = 5.438
```

全量拟合误差不是泛化指标。模型优劣仍应看前一节固定验证集结果；全量配置的目的只是模型确定后尽可能吸收已有官方观测。


## 第一章：顶层外层链路（get_bestshot 分发层）
### 代码结构说明
本项目采用分层决策架构，核心代码结构分为三层：
1. **顶层决策入口**：`main.py` 中的 `get_bestshot()` 函数作为总入口
2. **中层策略分发**：`Strategy_init()` 和 `Strategy_gote()` 分别处理先手和后手策略
3. **底层战术执行**：各单壶分支函数实现具体投壶逻辑

对于大多数定制需求，**您只需要修改 `main.py` 中的 `get_bestshot()` 函数**即可完成核心决策逻辑的调整，无需深入修改底层战术细节。
### 函数定义与作用
`get_bestshot()` 是AI机器人的核心决策入口函数，负责：
1. **阵营判断与分发**：根据 `self.player_is_init` 变量判断对局阵营类型
   - 先手阵营：调用 `Strategy_init()` 执行先手专属策略
   - 后手阵营：调用 `Strategy_gote()` 执行后手专属策略
### 关键特性
- 阵营适配：根据先手/后手身份自动切换策略逻辑

## 第二章：内层子策略统一链路（Strategy_init / Strategy_gote 共用同一架构）
1. 进入子策略入口函数后，定义局部变量 `shot_idx`，通过 `get_my_shot_index()` 获取当前本局待执行的投壶序号（1~8）
2. 获取当前冰面状态列表 `state_list`，包含所有冰壶的位置信息
3. 通过 match-case 匹配架构，根据 `shot_idx` 数值定向分发至对应序号的投壶分支：
   - 当 `shot_idx == 1` 时，进入第1壶独立执行分支
   - 当 `shot_idx == 2` 时，进入第2壶独立执行分支
   - 当 `shot_idx == 3` 时，进入第3壶独立执行分支
   - 当 `shot_idx == 4` 时，进入第4壶独立执行分支
   - 当 `shot_idx == 5` 时，进入第5壶独立执行分支
   - 当 `shot_idx == 6` 时，进入第6壶独立执行分支
   - 当 `shot_idx == 7` 时，进入第7壶独立执行分支
   - 当 `shot_idx == 8` 时，进入第8壶独立执行分支
4. 每个分支直接返回预设的BESTSHOT指令或调用战术函数生成指令
5. 若战术执行失败，自动返回默认投壶指令 `default_shot()`

## 第三章：单壶核心决策链路（最终执行层）
1. 进入对应序号的单壶执行分支后
2. 根据当前局面状态选择最优战术
3. 调用战术执行函数生成具体的投壶指令
4. 返回BESTSHOT格式的投壶指令

## 快速开始
### 运行方式
只需执行项目根目录下的 `main.py` 文件即可启动AI机器人：
```bash
python main.py
```

### 自动流程
1. 程序启动后会自动连接到本地测试服务器
2. 接收游戏状态并自动执行决策
3. 无需人工干预，全程自动完成投壶决策

### 注意事项
- 确保测试服务器已在本地7788端口启动
- 程序会自动处理先手/后手阵营切换
- 所有策略决策均由AI自动完成，无需人工输入

# 01 精读：AlphaZero: Mastering Chess and Shogi by Self-Play

## 论文定位

这篇论文把 AlphaGo Zero 的“搜索增强自我博弈”从围棋推广到国际象棋和日本将棋，核心问题不是某个棋类技巧，而是：在只有规则、没有人类棋谱、没有手工评估函数、没有开局库和残局库的条件下，是否能用同一套算法在不同完美信息博弈中达到超人水平。

对我们来说，它不是“照搬神经网络加 MCTS”的理由，而是一个训练范式证据：强策略不是靠裸策略网络直接从随机对局里硬学出来，而是靠搜索先把当前策略局部改强，再把搜索结果蒸馏回网络，形成循环。

## 方法细读

AlphaZero 使用一个双头网络 `f_theta(s) -> (p, v)`：

- `p` 是当前状态下各合法动作的先验概率。
- `v` 是当前局面的期望结果，取值可包括胜、负、平，不再只是假设二元胜负。
- 每一步用 MCTS 根据网络先验和价值评估做搜索。
- 搜索后得到根节点访问次数形成的改进策略 `pi`。
- 自我博弈用 `pi` 选动作，终局结果为 `z`。
- 训练样本为 `(s, pi, z)`。

损失函数由三部分组成：

```text
l = (z - v)^2 - pi^T log p + c ||theta||^2
```

这意味着网络同时学习两件事：价值头拟合自我博弈结果，策略头拟合搜索后的动作分布。关键是 `pi` 不是原始网络的输出，而是网络经过 MCTS policy improvement 之后的结果。

和 AlphaGo Zero 相比，AlphaZero 做了几处泛化：

- 不依赖围棋的旋转/翻转对称增强，因为国际象棋和将棋规则不具备同样对称性。
- 不维护“历史最佳模型”再做胜率门控，而是持续更新单一网络。
- 价值目标从纯胜率扩展为期望结果，能够处理平局。
- 搜索超参基本跨游戏复用，只对探索噪声按合法动作数缩放。

## 实验细读

论文报告 AlphaZero 在国际象棋、将棋、围棋三个任务中使用同一套算法设置。训练资源极大，使用大量 TPU 生成自我博弈和训练网络。结果显示：

- 国际象棋中数小时内超过 Stockfish。
- 将棋中数小时内超过 Elmo。
- 围棋中超过此前 AlphaGo Lee，并能与 AlphaGo Zero 旧版本对比。
- AlphaZero 搜索的节点数远低于传统 alpha-beta 引擎，但由于网络评估更集中，单位搜索质量更高。

实验的重点不是绝对资源，而是“搜索深度/宽度可以被学习到的先验替代一部分”。在棋类中，传统引擎搜索数量巨大，AlphaZero 搜索数量少很多，但每个展开点更有价值。

## 对数字冰壶的可迁移点

1. 搜索作为老师，网络作为学生。

   这与我们当前 `train_search_distill.py` 的方向一致。冰壶动作空间连续、结果随机，纯策略网络直接输出一个离散 tactic 容易受限。更稳的做法是先让搜索在当前局面下产生强动作或动作分布，再训练模型模仿。

2. 训练目标应该是分布，不只是 argmax。

   AlphaZero 学的是搜索访问次数分布 `pi`，不是单个最佳动作。对冰壶而言，相邻速度、角度、擦冰距离往往质量接近，学习分布比硬标签更能表达不确定性。

3. 价值最好预测得分分布或期望胜率。

   冰壶一局不是只有胜负，单壶和单 end 都有分差。我们现在用标量分数可以跑通，但更像论文路线的目标是 `[-8, 8]` 得分分布或考虑剩余 end 的胜率。

4. 搜索预算应集中在候选动作附近。

   AlphaZero 靠网络先验减少无效搜索。我们也应该用 tactic library 或模型先验给出候选，再做局部连续优化，而不是在全连续动作空间盲采样。

## 不能直接照搬的地方

AlphaZero 面对的是确定性、完美信息、离散动作游戏。数字冰壶有几个不同点：

- 动作是连续的：`v0, h0, w0` 以及擦冰距离。
- 转移有执行噪声，且官方服务器物理模型未知。
- 单壶回报可能受后续壶强烈影响。
- 擦冰是执行过程中的控制，不是单纯落子动作。

所以我们不能只做普通 MCTS。更合适的是 AlphaZero 式循环加 KR-UCT/连续动作局部搜索。

## 对当前项目的决策

短期：

- 保留搜索蒸馏路线。
- 继续让 `search_distill_robot.py` 使用模型选候选，再用 `continuous_shot_search.py` 做局部 refinement。
- 每次评估记录搜索分布、候选动作分数、最终动作和结果，避免只看胜率。

中期：

- 把训练标签从 tactic id 升级为搜索产生的动作分布。
- 增加 value head，输出分差或得分分布。
- 用历史模型池做自我博弈，避免只打 random/scripted。

长期：

- 如果官方服务器可大量调用，建立“官方物理校准模型 + 搜索老师 + 模型蒸馏学生”的闭环。

## 一句话结论

这篇论文支持我们的主路线：不要迷信裸强化学习，先用搜索把局部决策做强，再让模型蒸馏搜索。冰壶的区别是连续动作和噪声更核心，所以 AlphaZero 只能提供训练框架，不能提供动作搜索细节。

# 02 精读：AlphaGo Zero: Mastering Go without Human Knowledge

## 论文定位

AlphaGo Zero 是 AlphaZero 路线的原型。它证明在围棋中可以完全不使用人类棋谱，让程序通过自我博弈成为自己的老师。对我们而言，这篇比 AlphaZero 更强调“搜索目标如何进入训练循环”，也更清楚地说明了为什么 MCTS 不是推理阶段的附属模块，而是训练数据生成器。

## 方法细读

AlphaGo Zero 使用单个残差网络输出策略和价值：

```text
(p, v) = f_theta(s)
```

输入只包含棋盘上的黑白子和历史信息，不再使用人类特征。网络输出：

- `p`：每个落点的概率。
- `v`：当前玩家最终获胜的概率。

每一步执行 MCTS，搜索时每条边保存：

- `P(s, a)`：网络给出的先验概率。
- `N(s, a)`：访问次数。
- `Q(s, a)`：该动作下子树评估均值。

搜索选择边时最大化 `Q + U`，其中 `U` 由先验概率和访问次数决定。搜索结束后，根据根节点访问次数得到 `pi`，再用 `pi` 下棋。

训练数据为：

```text
(state, search_policy, final_winner)
```

网络目标是同时拟合搜索概率和最终胜负：

```text
l = (z - v)^2 - pi^T log p + c ||theta||^2
```

这里最重要的机制是“搜索概率通常强于原始网络概率”。也就是说，当前网络先指导搜索，搜索再反过来提供更强标签训练网络。

## 训练流程细读

论文流程可以抽象成四步循环：

1. 用当前网络自我博弈。
2. 每步用 MCTS 产生改进策略。
3. 保存 `(s, pi, z)`。
4. 训练网络逼近 `pi` 和 `z`。

AlphaGo Zero 还对比了监督学习网络。监督网络在人类走法预测上更好，但自我博弈网络实战更强。这一点对我们很重要：模仿已有脚本或 tactic library 可以作为冷启动，但不能成为上限。

## 实验细读

论文报告了几个关键结果：

- 从随机初始化开始，约三天训练产生 490 万局自我博弈。
- 每步 MCTS 约 1600 次模拟。
- 训练中 Elo 平滑上升，没有明显灾难性遗忘。
- 训练 72 小时后 100:0 击败曾击败李世石的 AlphaGo Lee。
- 人类棋谱监督学习更像人，但自我博弈策略更强。
- 残差网络和策略价值共享网络结构显著提高表现。

实验说明：自我博弈不是“随机互打然后等奇迹”，而是每一步都有搜索加强，搜索输出又被网络吸收。

## 对数字冰壶的可迁移点

1. tactic library 可作为冷启动，不应作为终点。

   我们目前使用战术库和搜索蒸馏，这相当于先用人造动作空间降低难度。后续必须允许连续优化动作超出战术库，否则会卡在候选动作质量上限。

2. 需要保存搜索分布。

   对冰壶，`pi` 可以是候选 tactic 的访问/评分分布，也可以是连续候选点经核回归后的权重分布。只存最佳动作会浪费很多教学信号。

3. 价值目标要从“赢/输”改为“得分/胜率”。

   单 end 冰壶的自然目标是得分分布 `[-8, 8]`。多 end 时还要引入剩余 end、当前分差和是否后手的胜率表。后面 Hammer shot 论文也支持这一点。

4. 网络架构先不必照抄 CNN。

   AlphaGo Zero 的棋盘结构很适合 CNN。我们的当前状态只有少量石头坐标，用 CPU 跑，MLP 可能更实用。等我们决定做 32x32 occupancy feature map 时，再考虑小 CNN。

## 风险和差异

AlphaGo Zero 的环境是确定且完全可信的。我们的最大风险是模拟器与官方服务器不一致。搜索如果基于错误 forward model，越搜可能越过拟合本地物理。因此：

- 官方服务器恢复后必须先做 shot calibration。
- 搜索结果需要加入噪声鲁棒性，不选“无噪声最优但稍偏就崩”的 shot。
- 擦冰必须被建模进动作，而不是只在发球后启发式补救。

## 对当前项目的决策

当前可执行改造：

- 在 evaluation trace 中保存候选动作分数分布。
- 蒸馏时把 label 从硬 tactic id 改成 soft distribution。
- 增加 value head 预测当前 end 的期望分。
- 自我博弈时保留历史模型快照作为 opponent pool。

暂不建议：

- 直接从零开始大规模 self-play。
- 直接上大 CNN。
- 忽略官方物理校准就把本地胜率当真实强度。

## 一句话结论

AlphaGo Zero 给我们的不是“用深网万能解决”，而是一个严密闭环：搜索改进策略，网络吸收搜索，更新后的网络再产生更好的搜索。我们要把这个闭环改造成适合连续、有噪声、有擦冰的冰壶版本。

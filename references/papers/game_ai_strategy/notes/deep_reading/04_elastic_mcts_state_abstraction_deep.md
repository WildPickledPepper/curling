# 04 精读：Elastic MCTS with State Abstraction for Strategy Game Playing

## 论文定位

这篇论文研究复杂策略游戏中的 MCTS 状态抽象。它不是冰壶论文，但对我们有一个重要提醒：搜索树太大时，可以把相似状态合并，提高采样效率；但抽象质量不稳定时，不能一直依赖抽象，否则根节点推荐策略可能变差。

“Elastic”的含义是：搜索前期使用状态抽象合并节点，后期放弃抽象并回到原始状态树，让搜索在压缩和精确之间切换。

## 方法细读

论文面对的是多单位策略游戏，动作空间随着单位数量组合爆炸。普通 MCTS 在这种场景中很难搜索有效深度。

作者使用近似 MDP homomorphism 做状态相似性判断。两个状态可合并的条件主要看：

- 奖励近似误差 `epsilon_R` 是否小于阈值。
- 转移近似误差 `epsilon_T` 是否小于阈值。

如果两个状态足够相似，就把它们放进同一个 abstract node，共享访问次数和累计奖励。

Elastic MCTS 的关键改造：

1. 搜索前期：按批次把 ground nodes 聚类成 abstract nodes，共享统计，减少树大小。
2. 搜索中期：继续使用抽象节点的统计做 selection 和 backpropagation。
3. 达到阈值 `alpha_ABS` 后：放弃抽象，把树展开回 ground nodes，用普通 MCTS 继续精细搜索。

这解决了一个矛盾：抽象能提升早期采样效率，但错误抽象会污染后期动作选择。

## 实验细读

实验平台是 Stratega 中的 Kill The King 策略游戏。比较对象包括：

- 规则 Combat Agent。
- 普通 MCTS。
- `MCTS_u`：带单位排序的 MCTS。
- `Elastic MCTS_u`：单位排序加弹性状态抽象。

主要结果：

- Elastic MCTS 在不同单位数量和地图布局下整体优于基线。
- 状态抽象能把树规模压缩约 10 倍。
- 抽象不是越久越好。论文实验显示，在搜索前 50% 或 75% 使用抽象效果较好，100% 全程抽象反而变差。
- 决策时间没有显著增加，说明抽象带来的额外计算没有抵消采样收益。

## 对数字冰壶的可迁移点

1. 冰壶也有“相似状态”。

   两个局面如果石头位置差异很小、可行战术和得分评估近似，那么搜索数据可以共享。尤其在执行噪声下，状态本来就是分布而非单点。

2. 但我们当前阶段不该优先做复杂状态抽象。

   现在最大误差来自物理模型和动作搜索，不是搜索树节点数量。过早做状态抽象可能把问题复杂化。

3. “先粗后细”非常适合连续动作搜索。

   这篇的思想可以迁移到动作层面：先用 tactic library 和粗采样缩小范围，再在局部连续参数上精搜。我们现在的 `continuous_shot_search.py` 已经类似这个策略。

4. 后期要回到精确状态。

   如果未来做状态聚类或缓存，不应在最终选 shot 时只依赖抽象估值。最后几轮候选必须用原始模拟器反复评估。

## 与当前项目的关系

可落地但优先级较低：

- 建立局面哈希或近邻缓存：相似局面复用候选 tactic 分数。
- 对搜索样本做 state embedding，训练 value 网络时增强泛化。
- 在 end 前期使用粗局面分组，最后几壶使用精确连续搜索。

暂不建议：

- 实现完整 Elastic MCTS。
- 大规模状态聚类。
- 用抽象节点直接替代最终动作评估。

## 一句话结论

这篇论文给我们的核心启发是“抽象只能做搜索加速，不能长期替代精确评估”。对冰壶当前阶段，更应把它理解为先粗后细的搜索预算管理思想。

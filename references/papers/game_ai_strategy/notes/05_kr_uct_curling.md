# 05 KR-UCT: Continuous Action Spaces with Execution Uncertainty

## 这篇解决什么问题

这是最贴近我们项目的核心论文之一。问题是：冰壶动作连续，且同一个目标动作会因为执行误差产生随机结果。传统做法先用领域知识列出少量候选动作，再对每个候选动作采样评估。但这样会浪费信息：一个动作的执行结果，其实也能说明附近动作的质量。

## 方法拆解

KR-UCT 的核心思想是用执行不确定性模型作为 kernel，把一次模拟结果泛化到相邻动作：

- 候选动作只作为初始化，不是最终动作全集。
- 每次执行样本不只更新原动作，也通过 kernel regression 影响相似动作的价值估计。
- kernel density 估计用于判断哪些连续空间区域探索不足。
- 搜索在 exploitation 和 exploration 之间做类似 UCB 的平衡。
- 这使算法能发现初始候选集之外的好动作。

论文特别强调：当好动作稀疏时，离散化候选集会严重依赖领域知识。KR-UCT 的价值在于把“动作执行噪声”从麻烦变成泛化信号。

## 实验和结论

实验在高保真冰壶模拟器上比较了 UCT、progressive widening、RAVE 等方法。KR-UCT 在 hammer shot 选择和更长搜索情境下都优于对照方法。论文还用大量重复模拟评估候选 shot 的期望得分，说明 KR-UCT 的优势来自更好的连续动作选择，而不只是偶然采样。

## 对我们数字冰壶的启发

这是下一版模型最该补的部分：

- 当前动作库只会选战术，不会连续微调。应该在选出 tactic 后，用 KR-UCT/局部 kernel search 调 `(v0, h0, w0, sweep)`。
- 我们本地代理要记录同一 shot 多次执行的落点分布，估计官方噪声 kernel。
- 擦冰也应进入连续动作：至少把 `sweep_distance` 当第 4 维，而不是固定启发式。
- 搜索结果可以继续蒸馏成轻量策略网络，在线才不会太慢。

## 局限

KR-UCT 需要比较可信的执行不确定性模型。若本地模拟器与官方服务器差异太大，kernel 学到的是假物理。官方服务器恢复后，第一任务应是校准 `(v, h, w, sweep) -> landing distribution`。

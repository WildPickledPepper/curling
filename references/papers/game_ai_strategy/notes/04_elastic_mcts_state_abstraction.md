# 04 Elastic MCTS with State Abstraction for Strategy Game Playing

## 这篇解决什么问题

Elastic MCTS 关注搜索树太大的问题。策略游戏中状态和动作组合爆炸，普通 MCTS 在有限时间内树太浅。状态抽象可以把相似节点合并，复用统计量，但抽象太粗又会误导搜索。论文提出“先合并提速，后拆开纠偏”的弹性机制。

## 方法拆解

算法维护一个状态到抽象组的映射。相似状态会被归为同一个抽象节点，共享访问次数和累计回报。相似性根据近似 homomorphism 思路判断，主要看：

- 相同行动下奖励差异是否小于阈值。
- 转移结果差异是否小于阈值。

Elastic 的关键是有一个 MCTS 迭代阈值：前期使用抽象加快探索，到达阈值后放弃抽象，把组统计分配回具体节点，让后期搜索回到 ground state 精修，避免抽象误差持续污染最终推荐动作。

## 实验和结论

在 Stratega 策略游戏场景中，Elastic MCTS 显著优于普通 MCTS，并减少约一个数量级的树大小。结论不是“抽象越多越好”，而是抽象应该服务于早期探索，后期必须允许细分。

## 对我们数字冰壶的启发

冰壶状态也可以做战术抽象：

- 大本营最近壶归属、得分差、前卫壶数量、中心线附近阻挡、可击打目标数量。
- 前几壶可以按战术形态合并，快速判断“进攻/防守/清障/保护”。
- 临近最后几壶，尤其 hammer shot，必须回到具体坐标和连续动作优化。

这和我们的模型路线很契合：先用 tactic-level policy 缩小动作，再用连续搜索微调 `(v, h, w, sweep)`。

## 局限

论文场景不是物理连续动作游戏。它的抽象判断依赖可查询 forward model 和奖励/转移误差。我们要先有足够可信的本地模拟器，否则抽象合并可能把官方服务器上完全不同的局面混为一谈。
